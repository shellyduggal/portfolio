Case Study:
Python Text Adventure

I created this text adventure game for the console as a course assignment. At the time, I was very new to coding and simply had tons of fun going through all of the logic to make the game work. Now that I know a bit more, it's nice to see how far I've come.

Find the code on my Github repo

The Challenge

The assignment was simply to use Python to create a text adventure game similar to the popular game "Zork"

At this point in the course, we were just learning about functions and programming logic. The difficulty in the assignment came from keeping the functions organized to ensure that there were no loose ends in the game.

The Process

In order to unwrap the complexity of the game, I started out with a pad and pencil. I wrote out the story I wanted to tell, along with the different paths a character should be able to take. I didn't start coding until I had a plan.

By the time I finished the code and tested all of my options, it was 3 am! My poor husband was very supportive when I woke him in the middle of the night to show him my first app - he even obliged me by groggily playing the game

Next Steps

I'd like to give my game a face

After diving deeper into the world of web development, I'd like to revamp my first app. I think it would be nice to make a browser-friendly version so that I can share it with friends who are not comfortable playing it in the console.

I'm very interested in single page architecture and this feels like a good choice for a first project. I recently started learning React.js and hope to use it to bring my game to the web
